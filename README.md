# Swiggy SQL Analysis — Task 3 Submission

This repository contains the SQL queries prepared for the Swiggy case study.  
All queries are taken exactly as written in the reference project (swiggy.pdf),  
without any modification to table names, structure, or logic.

---

##  Files Included

### **queries_task3_exact.sql**
Contains the exact SQL queries from the Swiggy analysis, covering:
- Customers located in Delhi  
- Average restaurant ratings in Mumbai  
- Identification of active customers  
- Total orders per customer  
- Revenue generated by each restaurant  
- Top 5 restaurants by average rating  
- Customers who never placed an order  
- Orders by customers from Mumbai  
- Orders placed in the last 30 days  
- Delivery partners with more than one delivery  
- Customers ordering on exactly three different days  
- Delivery partner with the highest number of unique customers  
- Same-city customers ordering from the same restaurant on different dates  

### **raw_data.txt**
Explains the table structure referenced in the queries.

### **presentation.txt**
Summarizes the insights and key findings derived from the SQL analysis.

---

##  Purpose of the Repository
This project demonstrates SQL-based data analysis on a food delivery platform scenario.  
The queries focus on customer behaviour, restaurant performance, delivery partner efficiency,  
and real-world analytics using joins, grouping, filtering, and aggregation techniques.
